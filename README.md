# Trust-Aware Quantum-Assisted Digital Twin Control for Secure ISAC in 6G Open RAN

Reference simulator and experiment data for the paper:

> **Trust-Aware Quantum-Assisted Digital Twin Control for Secure and Adaptive ISAC in 6G Open RAN**
> Yassir Ameen Ahmed Al-Karawi, *submitted to IEEE Journal on Selected Areas in Communications (JSAC)*, 2026.

---

## Overview

This repository contains the discrete-time simulator used to produce every numerical result in the paper. It implements the full closed-loop control stack described in the manuscript:

- **Physical layer** — Rician fading with AR(1) temporal correlation, log-normal shadowing, $8{\times}8$ UPA steering, mmWave path loss at 28 GHz
- **Precoding** — coordinated joint-transmission regularised zero-forcing (JT-RZF) across four cooperating BSs (256-antenna aggregate array), with MRT and ZF fallbacks
- **Sensing** — Swerling-I detection with coherent pulse integration (512-chirp CPI), CRLB-based accuracy proxy, AR(1) correlated clutter, dedicated 400 MHz sensing waveform
- **Mobility** — pedestrian Rayleigh velocities with per-slot position innovation
- **Digital twin** — Kalman-like filtering with configurable synchronisation delay, fidelity metric coupled to state mismatch
- **Trust process** — Bayesian-EWMA update with bounded log-likelihood and anomaly evidence
- **Quantum-assisted screening** — classical surrogate of the variational scoring described in the paper (deterministic, trainable online)
- **Trust-aware gate** — convex blend of optimal and safe-fallback actions, hard fallback below the trust floor
- **Anomaly injection** — Poisson-onset, geometrically-distributed jamming, spoofing, and mixed episodes

## Quick start

```bash
# 1. Clone the repository
git clone https://github.com/YassirALKarawi/trust-aware-isac-sim.git
cd trust-aware-isac-sim

# 2. Install dependencies (Python 3.10+)
pip install -r requirements.txt

# 3. Run a single smoke test (≈ 30 seconds)
python src/controller.py

# 4. Reproduce the baseline comparison from the paper (≈ 3 minutes)
python src/run_baseline.py

# 5. Run the full experiment suite (≈ 30 minutes)
python src/run_all.py
```

Results are saved as JSON files under `results/`. Pre-computed results from the paper are included in this repository.

## Reproducibility

All experiments are seeded from a single `master_seed` parameter in `config.py` (default: `20260417`). Monte Carlo realisations are generated by deterministic seed offsets (`master_seed + 1000·mc_index`). Rerunning any experiment from the same seed produces bit-identical results.

The simulator runs on CPython 3.10+ with NumPy ≥ 1.24, SciPy ≥ 1.10, and Matplotlib ≥ 3.7. No GPU or quantum-hardware dependency is required; the quantum-assisted screener is a classical deterministic surrogate (see `src/screening.py`) as described in Section IV-C of the paper.
